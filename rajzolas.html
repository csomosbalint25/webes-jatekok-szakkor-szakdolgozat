A játék célja

Biztosan jól ismered a pontösszekötős rajzokat, ahol a számozott pontok összekötésével lehet befejezni egy rajzot. Ebben a játékban ilyen rajzokat készítünk, de nem a hagyományos módon.

Interaktív példa

Útmutató

Az útmutató a Visual Studio Code fejlesztői környezethez készült.

HTML

Minthogy webes játékokról van szó, a Visual Studio Code alkalmazás megnyitása után elsőként létre kell hozzunk egy weboldalt index.html néven, ahol a játékot fogjuk elkészíteni. Ehhez az alap HTML-struktúrát fogjuk használni, amit könnyedén előhozhatunk a ! emmet paranccsal és az ENTER-rel. Ekkor az alábbi kódot kapjuk:

                
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
    
</body>
</html>

Állítsuk át az oldal nyelvét magyarra (<html lang="hu">), a weboldal címét pedig a játék nevére (<title>Kösd össze a pontokat</title>).

Ebben a feladatban (és a későbbi feladatok során is) a HTML-oldalon az alábbi elemeket fogjuk elhelyezni:

  • játék neve
  • vászon
  • vászon stílusa
  • JavaScript-kód

Ezeket a <body></body> tagek (vagyis címkék) között fogjuk elhelyezni. Kezdjük is a címmel, ami egy 1-es szintű címsor lesz:

                
<body>
    <h1>Kösd össze a pontokat!</h1>
</body>

A böngészőablak frissítése után megjelenik a "Kösd össze a pontokat!" szöveg.

A cím alatt helyezkedik el a <canvas></canvas>, vagyis a vászon, melynek szélessége legyen 313 pixel, magassága 407 pixel. A <canvas> tag id attribútuma egyedi azonosító megadására szolgál. Később a JavaScript fájlban ezzel az azonosítóval hivatkozhatunk a vászonra.

                
<body>
    <h1>Kösd össze a pontokat!</h1>
    <canvas width="313" height="407" id="vaszon"></canvas>
</body>

Ebben a pillanatban szabad szemmel még nem látható az eredmény, de a vászon már rendelkezésre áll a megadott helyen és méretben. Annak érdekében, hogy a vászon meglétét ellenőrizhessük, lássuk el a vásznat vékony fekete szegéllyel az alább látható módon:

                
<body>
    <style>
        canvas{
            border: 1px solid black;
        }
    </style>
    <h1>Kösd össze a pontokat!</h1>
    <canvas width="313" height="407" id="vaszon"></canvas>
</body>

Ez a vászon lehetőségek tárházát nyitja meg előttünk a játékfejlesztéshez. Azonban a vászonra csak JavaScript kódot használva tudunk rajzolni (és később játékot írni), ezért ehhez a vászonhoz hozzá kell rendelni egy kódot, vagyis scriptet. A kód forrásaként az index.js fog szolgálni:

                
<body>
    <style>
        canvas{
            border: 1px solid black;
        }
    </style>
    <h1>Kösd össze a pontokat!</h1>
    <canvas width="313" height="407" id="vaszon"></canvas>
    <script src="index.js"></script>
</body>

Vigyük az egérmutatót az index.js szöveg (link) fölé, majd a felugró ablakban kattintsunk a Follow link opcióra, hogy elérjük ezt a JavaScript fájlt. Miután rákattintottunk, egy figyelmeztetést kapunk, hogy az index.js fájl nem létezik. Valóban nem létezik még ez a fájl, de a Create File opcióra kattinva a Visual Studio Code létrehozza és megnyitja nekünk ezt a fájlt.

Ebben a projektben nem egy, hanem két vásznunk (<canvas>) lesz az egyes rajzokhoz. Ennek értelmében a végleges HTML-kód a következő lesz:

                
<!DOCTYPE html>
<html lang="hu">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Kösd össze a pontokat</title>
</head>
<body>
    <style>
        canvas {
            border: solid 1px black;
        }
    </style>
    <h1>Kösd össze a pontokat!</h1>
    <canvas width="313" height="407" id="vaszon1"></canvas>
    <canvas width="469" height="350" id="vaszon2"></canvas>
    <script src="index.js"></script>
</body>
</html>

JavaScript (JS)

Vászon elérése

A canvasre rajzolás nem más, mint parancsok sorozatának alkalmazása a canvasre. Ehhez viszont valahogyan el kell érnünk a canvast, vagyis a HTML dokumentumból ki kell választanunk a <canvas> elemet, és azt el kell tárolnunk egy konstansban, hogy később könnyen tudjunk rá hivatkozni. Azért konstansban tároljuk el, mert ez a vászon nem fog változni a játékfejlesztés során, vagyis mindig ugyanzt a vásznat fogjuk használni. A JS kód első két sora tehát a következő lesz két különböző vászon eltárolása esetén:

                
/*Canvas elérése*/
const canvas1 = document.querySelector('#vaszon1');
const canvas2 = document.querySelector('#vaszon2');

A const kulcsszó vezeti be azt, hogy egy konstanst hozunk létre, majd a konstans elnevezése után ("canvas1", "canvas2") rögtön egy = jellel értéket adunk neki a querySelector() HTML DOM metódussal. Ez az érték a HTML dokumentum #vaszon1 és #vaszon2 egyedi azonosítókkal ellátott eleme, vagyis a <canvas> tagek.

💭 Megjegyzés

A const kulcsszóval olyan változókat hozhatunk létre, melynek értéke nem változik a program futása során.

A <canvas> csak egy rajzfelület, önmagában még nem ad rajzoló eszközöket a kezünkbe. Annak érdekében, hogy erre a vászonra köröket, téglalapokat, vonalakat, görbéket, képeket és szöveget tudjunk rajzolni különböző színekben, méretekben, elforgatással és egyéb transzformációkkal, le kell kérnünk a vászon 2D kontextusát a getContext() HTML DOM metódussal. Ez a kontextus biztosítja a játékfejlesztéshez használt rajzolási műveleteket, pl. ctx1.drawImage() képkirajzoláshoz, ctx1.beginPath() alakzatrajzolás indításához vagy ctx1.arc() körív rajzolásához. Ezt a két kontextust ctx1 és ctx2 néven tároljuk el.

                
/*Canvas elérése*/
const canvas1 = document.querySelector('#vaszon1');
const ctx1 = canvas1.getContext('2d');

const canvas2 = document.querySelector('#vaszon2');
const ctx2 = canvas2.getContext('2d');

Háttérkép

A mintaként szolgáló képeket a canvas háttérképeként fogjuk használni, ezért kellett a vászon méretét a háttérképekével megegyezőre beállítani. Ha a vászon szélességének és magasságának aránya eltér a háttérkép arányaitól, akkor a háttérkép transzformálva (nyújtva) lesz kirajzolva. De hogyan lehet képet beilleszteni?

A képet el kell tárolnunk valahol, így egy konstanst hozunk létre, mely egy kép lesz, majd ennek a képnek megadjuk a forrását, vagyis a kép elérési útvonalát:

                
/*Képek betöltése*/
const giliszta = new Image();
giliszta.src = 'worm.png';

Ezt a képet ki kell rajzolnunk a vászonra, amit az alábbi kóddal tehetünk meg. Az első paraméter a kirajzolni kívánt kép (melyet a giliszta konstansban tároltunk el), a második és harmadik paraméter a kép bal felső sarkának x és y koordinátája, a negyedik és ötödik paraméter pedig a kép szélessége és magassága pixelben. Ezt a képet, viszont, csak akkor rajzolhatjuk ki, ha a képek már betöltöttek.

                
giliszta.onload = function() {
    ctx1.drawImage(giliszta, 0, 0, 313, 407);
};

Feladat ✅
Ugyanezt a műveletsort hajtsuk végre a másik vásznon a másik forrásképpel!

Megoldás 
                    
const hal = new Image();
hal.src = 'fish.png';
hal.onload = function() {
    ctx2.drawImage(hal, 0, 0, 469, 350);
};

A képek betöltése után a következőt kell látnunk.

Egérmutató pozíciójának kiírása

A pontok összekötéséhez fontos információ, hogy hol helyezkednek el a pontok. Úgy is fel lehetne tenni a kérdést, hogy mi a pontok x és y koordinátája a vásznon. Milyen jó lenne, ha ehhez az információhoz mindössze azáltal jutnánk hozzá, hogy a pontok fölé visszük az egeret! Hogyan lehet lekérdezni és kiírni az egérmutató helyzetét?

A következőkben megismerkedünk az egérmozgással, mint eseménnyel.

Mi számít eseménynek? Az egérmozgáson kívül ide tartozik többek között az kattintás és a különböző billentyűk lenyomása vagy felengedése is. A későbbi játékok során találkozni fogunk ezekkel is, de most az egérmozgás eseményre lesz szükségünk. Az imént felsorolt események bármikor megtörténhetnek, azonban alapvetően nincs hatásuk. Ha azt szeretnénk, hogy valamilyen hatásuk legyen, akkor egy eseményfigyelőt kell hozzárendelni valamelyik elemhez (mi a weboldalhoz fogjuk hozzárendelni), amelynél definiálhatjuk, hogy milyen eseményt figyeljen, és ha az bekövetkezik, akkor mi történjen. Ennek általános formája az alábbi:

                
elem.addEventListener(esemény, függvény);

Az első paraméter határozza meg az esemény típusát (egérmozgás/mousemove, kattintás/click, billentyűlenyomás/keydown, billentyűfelengedés/keyup, weboldal vagy kép betöltése/onload stb.), a második pedig azt a függvényt, amit az esemény bekövetkeztekor meghívunk, vagyis azt mondja meg, hogy mi történjen egy adott esemény bekövetkezésekor. Egy elemhez több eseménykezelőt is rendelhetünk (akár ugyanolyan típusúakat is anélkül, hogy azok felülírnák egymást).

                
document.addEventListener('mousemove', function(e) {})

A fenti kód még nem teljes, hiszen a {} jelek között meg kell írnunk az egérmutató vízszintes és függőleges helyének eltárolását. Matematikából már tudjuk, hogy egy koordinátarendszerben a vízszintes tengely jele az x, a függőleges tengelyé y. A vászon felülete is egy koordinátarendszer, viszont itt az origó nem középen van, hanem a bal felső sarokban. Ebből a pontból az x tengely balra nő, jobbra csökken, viszont az y tengely a megszokottól eltérően lefelé nő, felfelé csökken. Az imént említett koordinátákkal az egérmutató is rendelkezik, így nincs más dolgunk, mint lekérdezni ezeket a koordinátákat.

Elsőként (még az eseménykezelő függvény előtt) hozzuk létre az egérmutató koordinátáit eltároló x és y nevű változókat, melyeknek adjuk a 0 kezdőértéket! Változót a let kulcsszóval hozhatunk létre:

                
let x = 0;
let y = 0;
document.addEventListener('click', function(e) {})

Ezután ezeknek a változóknak adjuk értékül az egér x és y koordinátáit. A függvény e paramétere jelöli az egérmutatót, melynek vízszintes és függőleges koordinátáját a .offsetX és .offsetY tulajdonságok tartalmazzák:

                
let x = 0;
let y = 0;
document.addEventListener('click', function(e) {
    x = e.offsetX;
    y = e.offsetY;
})

Most már az x és y változók tartalmazzák, hogy hol van az egérmutató.

Hogy számunkra is egyértelmű legyen az egérmutató vízszintes és függőleges pozíciója, írjuk ki azt a vászonra. A szövegkiírás menete a következő lépésekből áll:

  • betűméret (pixelben) és betűtípus beállítása (.font),
  • betűszín megadása (.fillStyle), és
  • a szöveg kiírása (.fillText(mit, hovaX, hovaY)). A hovaX és hovaY paraméterek mondják meg, hova kerüljön a szöveg bal alsó sarka.
                
/*Egérmutató pozíciójának kiírása*/
let x = 0;
let y = 0;
ctx1.font = '20px Arial';
ctx1.fillStyle = 'black';
ctx1.fillText('(' + x + ',' + y + ')', 10, 50);

/*Egérmutató X és Y koordinátájának lekérdezése és eltárolása*/
document.addEventListener('mousemove', function(e) {
    x = e.offsetX;
    y = e.offsetY;
})

A fenti kód folyamatosan frissíti az x és y változókat, mégis csak a (0,0) koordinátapár látható a canvas bal felső sarkában, hiába mozgatjuk az egeret a canvas fölött. Miért van ez? A programkód szekvenciálisan, vagyis sorról sorra fut le. Tehát az x és az y változók értékét beállítjuk 0-ra, majd rögtön kiírjuk azt és csak ezután kezdjük frissíteni a változók értékét az egérmutató aktuális pozíciója alapján. Könnyen láthatjuk, hogy a kiírásnak többször meg kell történnie, hogy mindig a legfrissebbet írja ki. Mi lenne, ha a kiírást egy függvénybe helyeznénk el, ami utolsó lépésként önmagát hívja meg, vagyis folyamatosan kiírja az x és y változók értékét.

                
/*Egérmutató pozíciójának kiírása*/
let x = 0;
let y = 0;
function következő() {
    ctx1.font = '20px Arial';
    ctx1.fillStyle = 'black';
    ctx1.fillText('(' + x + ',' + y + ')', 10, 50);
    requestAnimationFrame(következő);
}
következő();

/*Egérmutató X és Y koordinátájának lekérdezése és eltárolása*/
document.addEventListener('mousemove', function(e) {
    x = e.offsetX;
    y = e.offsetY;
})

A fenti kódban a requestAnimationFrame(következő); lesz az a sor, ami újrahívja a következő függvényt. A requestAnimationFrame() megmondja a böngészőnek, hogy egy animációt szeretnénk elindítani, ezért ütemezi a következő képkockát. Olyan ez, mint az ujjunkkal pörgethető, rövid történeteket bemutató könyvecskék, melyek minden oldala egy picit másabb az előzőnél, ezért az "animáció" csak gyors pörgetés esetén látványos. A requestAnimationFrame() paraméterébe azt a függvényt kell írnunk, ami alapján az újrarajzolást el szeretnénk végezni. A mi esetünkben az újrarajzolás, vagyis a következő képkocka a következő függvényben meghatározottak szerint lesz más, ezért ezt a függvényt kell meghívnunk. Erről bővebben olvashatsz a W3Schools oldalán.

A mi kódunk jelenleg annyiban működik másképp az imént említett könyvecskéknél, hogy nem tűnnek el a korábban kirajzolt képkockák, hanem egymásra rajzolódnak...

Látható, hogy még nem működik tökéletesen. Az is látszik, hogy mi a gond: az újabb és újabb kiírások egymásra kerülnek, ezért minden kiírás alatt látszik az összes korábbi koordináta-pár. Kellene egy "radír eszköz" a kezünkbe, amivel a kiírás előtt letöröljük a vászon bal felső sarkát minden kiírás előtt. Mivel minden kiírást egy törlés előz meg, így a törlést is a függvényben kell elhelyezni a kiírás elé az alábbi módon.

                
/*Egérmutató pozíciójának kiírása*/
let x = 0;
let y = 0;
function következő() {
    ctx1.clearRect(0, 0, 140, 60);  //törlés téglalap alakban
    ctx1.font = '20px Arial';
    ctx1.fillStyle = 'black';
    ctx1.fillText('(' + x + ',' + y + ')', 10, 50);
    requestAnimationFrame(következő);
}
következő();

/*Egérmutató X és Y koordinátájának lekérdezése és eltárolása*/
document.addEventListener('mousemove', function(e) {
    x = e.offsetX;
    y = e.offsetY;
})

A törlés művelet a képkirajzoláshoz hasonló paraméterekkel rendelkezik:

  1. a törlendő terület bal felső sajkának x pozíciója pixelben
  2. a törlendő terület bal felső sajkának y pozíciója pixelben
  3. a törlendő terület szélessége pixelben
  4. a törlendő terület magassága pixelben

Az imént, tehát, egy 140px széles és 60px magas téglalap alakban töröltünk a canvas bal felső sarkától kezdve, hogy az alábbi eredményt kapjuk, amikor a giliszta bal felső pontja fölé visszük az egeret:

Töröttvonal rajzolása

Jöhet a vonalak rajzolása!

Vonalat (törött vonalat) rajzolni a ctx.beginPath(); paranccsal lehet, mely jelzi a vászonnak, hogy vonalrajzolás kezdődik. Ezután meg kell mondanunk a ctx.moveTo(kezdoX, kezdoY); parancssal a kezdőpontot, ahonnan az első vonalat fogjuk húzni. Ezután minden további vonalat a ctx.lineTo(kovetkezoX, kovetkezoY); utasítással tudunk megadni. Az első két pont közötti vonal meghatározása a gilisztát ábrázoló vásznon így néz ki:

                
/*Vonalak megrajzolása*/
ctx1.beginPath();
ctx1.moveTo(72, 147);
ctx1.lineTo(3, 236);

Tényleges vonalat azonban még nem látunk, mivel nem rendeltünk stílust ehhez a vonalhoz. Körvonalat és kitöltést mindig a legutóbb definiált alakzathoz tudunk a kör körvonalának definiálásához hasonló módon:

                
/*Megrajzolt alakzat körvonalának definiálása*/
ctx1.lineWidth = 1;
ctx1.strokeStyle = 'black';
ctx1.stroke();
